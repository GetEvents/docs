# URL de production de GetEvent

Ce document centralise les adresses publiques utilisées par le projet GetEvent.

| Service | URL |
| --- | --- |
| Application frontend | [https://getevent.netlify.app/](https://getevent.netlify.app/) |
| API backend | [https://getevent-back.onrender.com](https://getevent-back.onrender.com) |
| Documentation Swagger | [https://getevent-back.onrender.com/api-docs](https://getevent-back.onrender.com/api-docs) |
| Document OpenAPI JSON | [https://getevent-back.onrender.com/api-docs.json](https://getevent-back.onrender.com/api-docs.json) |
| État de l’API | [https://getevent-back.onrender.com/health](https://getevent-back.onrender.com/health) |

## Préfixes des routes API

Les routes métier utilisent les préfixes suivants :

- Authentification : `https://getevent-back.onrender.com/api/auth`
- Événements : `https://getevent-back.onrender.com/api/events`
- Participations : `https://getevent-back.onrender.com/api/participations`
- Paiements : `https://getevent-back.onrender.com/api/payments`
- Messages : `https://getevent-back.onrender.com/api/messages`
- Notifications : `https://getevent-back.onrender.com/api/notifications`

Les routes détaillées, leurs paramètres et leurs réponses sont disponibles dans la documentation Swagger.
